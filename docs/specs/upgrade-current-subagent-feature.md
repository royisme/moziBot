│ 当前 mozi 的 subagent 路径是同步阻塞式调用：src/runtime/tools.ts 中的 createSubagentTool() 调用 src/runtime/subagent-registry.ts 的 SubagentRegistry.run()，后者直接 await agent.prompt() 并返回最终文本。这会导致 agent               │
│ 虽然可以声称“已经分配 subagent”，但系统层并没有真正建立一个可管理、可监控、可在完成后主动通知父会话/用户的后台子任务生命周期。                                                                                                         │
│                                                                                                                                                                                                                                        │
│ 代码库里已经存在另一套更接近目标形态的基础设施：src/runtime/host/message-handler.ts 的 MessageHandler.startDetachedRun() 与 src/runtime/host/message-handler/services/run-lifecycle-registry.ts 的 RunLifecycleRegistry，以及一组 host │
│  session subagent 文件：src/runtime/host/sessions/spawn.ts、src/runtime/host/sessions/subagent-registry.ts、src/runtime/host/sessions/subagent-announce.ts。这些能力与 OpenClaw 的 spawn/register/announce 模型接近，但尚未接到 agent  │
│ 可用的 subagent_run 工具主链路上。                                                                                                                                                                                                     │
│                                                                                                                                                                                                                                        │
│ 本次改造目标是把 mozi 的 subagent 能力升级为真正的后台子任务系统：主 agent 能真实 spawn subagent，系统能管理运行状态，用户/父会话能监控进度，并在完成、失败、中止、超时时自动收到结果通知。                                            │
